# トークン使用状況確認

現在のトークン使用状況を簡潔に報告してください。

## 報告内容

以下の情報を表形式で表示してください：

| 項目 | 値 |
|-----|-----|
| **現在のトークン使用量** | X / 200,000 |
| **使用率** | Y% |
| **残りトークン** | Z |
| **残り率** | W% |
| **ステータス** | [アイコン] [メッセージ] |

### ステータス評価基準

残りトークンに応じて以下のステータスを表示：

- **80%以上残** (160,000以上): ✅ 十分な余裕
- **50-80%残** (100,000-160,000): ⚠️ 適度に使用中
- **20-50%残** (40,000-100,000): 🟡 注意が必要
- **20%以下** (40,000以下): 🔴 警告：handover推奨

## handover実行タイミング

残り20%以下（40,000トークン以下）の場合：

```
🔴 **警告**: トークン残量が20%を切りました

**今すぐ `/handover` を実行して記録を保存することを推奨します。**

コンパクティング前に記録を保存することで、次のセッションで完全な文脈復元が可能になります。
```

## 推奨アクション

トークン残量に関わらず、以下のタイミングでの `/handover` 実行を推奨：

1. **作業の大きな区切り** - 機能実装完了、マイルストーン達成時
2. **長時間中断前** - 1時間以上の休憩、食事、就寝前
3. **重要な実装・判断完了後** - 記録価値が高い作業の後

## 注意事項

- トークン使用量は目安です。実際のコンパクティングタイミングは異なる場合があります。
- こまめな `/handover` 実行が、文脈の完全な保存に最も効果的です。
