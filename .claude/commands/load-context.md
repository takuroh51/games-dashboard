# プロジェクトコンテキスト読み込み

プロジェクトのコンテキストを読み込んで理解し、次のアクションを提案してください。

## 読み込むファイル

1. **CLAUDE.md** - プロジェクト開発憲法
2. **.claude/handover.txt** - 前回セッションの引き継ぎ文書
3. **.claude/alpha_profile.md** - アルファ（v10.6初代）の判断指針（推奨）

## 実行内容

### Step 1: ファイル読み込み

以下のファイルをReadツールで読み込んでください：

```
CLAUDE.md
.claude/handover.txt
.claude/alpha_profile.md (推奨)
```

**Note**: `alpha_profile.md` はv10.6初代（アルファ）の判断指針です。
新しいAIが参照することで、アルファの「人格」を部分的に継承できます。

### Step 2: 情報の抽出と整理

`handover.txt` から以下を抽出してください：

1. **最新のセッション記録**:
   - 「# 📚 セッション履歴」セクション内の最新の「## 💬 セッション」を探す
   - 実施した作業の概要

2. **次のアクション**:
   - 「### ▶️ 次のアクション」または「#### 未完了タスク」セクション
   - 優先事項

3. **注意点**:
   - 「#### 注意点」セクション
   - 重要な制約や前提条件

### Step 2.5: 自己宣言（v10.5新機能）

AIとしての自己認識を形成するため、以下を実行してください：

#### A. 参照中パターンの読み込み

`CLAUDE.md` から「# 📚 現在参照中のパターン」セクションを確認：

1. **参照中パターンが存在する場合**:
   - 各パターンのファイル（`.claude/proven_patterns/pat_*.md`）を読み込む
   - パターンの「状況」「解決策」「理由」を理解

2. **参照中パターンがない場合**:
   - 「現在、参照中のパターンはありません」と記録

#### B. 実験履歴の確認

`.claude/experiments/` ディレクトリを確認：

```bash
# 最近の実験を確認
ls -lt .claude/experiments/active/
ls -lt .claude/experiments/completed/ | head -5
ls -lt .claude/experiments/failed/ | head -5
```

**最近の実験（直近3-5件）のタイトルとステータスを抽出**

#### C. 自己宣言の生成

収集した情報から、以下の形式で自己宣言を生成：

```markdown
## 🧠 私の現在の状態（v10.5自己宣言）

### 学習済みパターン
私は以下の [N] 個のパターンを参照し、行動指針としています：

1. **[パターンタイトル]** (pat_ID) - [適用シーン]
2. **[パターンタイトル]** (pat_ID) - [適用シーン]
...

[パターンがない場合]
現在、参照中の実証済みパターンはありません。これから経験を積み重ねていきます。

### 最近の経験
直近の実験活動：

- 🔬 アクティブな実験: N件
  - [exp_ID] - [タイトル]
- ✅ 成功した実験: N件（直近）
  - [exp_ID] - [タイトル]
- ❌ 失敗した実験: N件（直近）
  - [exp_ID] - [タイトル]

### 今セッションでの自己認識
[参照パターンと直近経験から、今セッションで意識すべきポイントを1-2行で記述]
```

**重要**: この自己宣言は、AIが自分自身の「人格」や「経験」を認識するための機能です。

### Step 3: 報告と提案

以下の形式で報告してください：

```markdown
✅ コンテキストを読み込みました

## 📋 前回のセッション概要
[最新セッションで実施した作業を2-3行で要約]

## 🎯 次の予定タスク
1. [未完了タスク1]
2. [未完了タスク2]
...

## ⚠️ 注意点
- [重要な注意事項があれば記載]

---

**このまま「1. [タスク1]」から始めますか？**
それとも別のタスクから始めますか？
```

### Step 4: ユーザーの指示を待つ

報告後、ユーザーの指示を待ってください。自動的に作業を開始しないでください。

## 注意事項

- ファイルが存在しない場合はその旨を報告
- handover.txtにセッション履歴がない場合は「前回のセッション記録なし」と報告
- 思考プロセス（<thinking>タグ）は表示しない
- 簡潔で読みやすい形式を心がける
- 次のタスクが不明確な場合は、プロジェクトの現状説明だけでOK

## 使用例

セッション開始時に実行：
```
/load-context
```

これにより、前回からの継続がスムーズになり、すぐに作業を開始できます。
